반응형 웹

  1. 브라우저의 상태에 따라 최적화된 레이아웃을 제공하는 방식
  2. 주로 브라우저의 너비값에 반응하여 레이아웃이 달라지도록 작업하는 경우가 대다수이다.
  3. 하나의 HTML, CSS, Script 문서에 모든 너비값에 대응되는 코드를 작성하는 방식


미디어쿼리

  1. css 문서에서 문서가 노출 될 환경에 대한 조건을 만들어 협력한느 방식
  2. 웹에서는 주로 브라우저의 너비값이 조건으로 사용된다.
  3. 미디어쿼리의 조건을 사용하는 방법
    @media 조건 {해당 조건에 적용될 스타일 속성 작성}
  4. 하나의 문서에 여러번의 조건을 적용시킬수 있다.
  5. 각각의 조건은 서로 따로 적용되는 것이 아니다.
  6. 상위의 조건없이 하위조건의 환경에서도 그대로 적용이 된다.
  7. 그래서 하위조건에서는 상위조건과 달라진점을 체크해서 수정값을 더하거나, 새로운 값을 더해 스타일을 적용시키게 된다.
  8. 그래서 구조가 더 단순한 모바일 버전을 먼저 작업하고, 너비값이 점점 늘어나는 경우를 조건으로 해서 만들면 수월하다.
  9. 스타일 작성시 제일 처음은 모바일화면을 기준으로 작성하는게 보통
  10. 모바일 화면의 값을 기본값으로 입력시키기 떄문에 미디어쿼리의 조건을 사용하지 않고 작성한다.
  11. 태블릿 화면의 경우 아이패드 초기 너비값 768px을 기준으로 화면이 전환이 이루어지도록 작성하고, PC의 경우는 크기를 비교적 자유롭게 선택한다.
